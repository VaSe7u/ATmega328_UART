38 ### Building and uploading using one of the provided *`Makefile`s* :page_with_curl:
39 > _Prerequisites: AVR-GCC and Make_
40 
41 #### Building **`ATmega328_UART`** to a static library
42  - Open *`.../ATmega328_UART/Makefile`* and edit the variables: `$HOST_PLATFORM`, `$F_CPU` and `$UART_BAUD`.
43  - Execute `> make` inside *`.../ATmega328_UART/`* to recompile the library according to the specified frequency and baud rate.
44 > :grey_exclamation: *The library must be recompiled every time `$F_CPU` and/or `$UART_BAUD` variables are changed!*
45 
46 #### Building and uploading an example
47  - Open the *`Makefile`* that is in the root directory of the chosen example _(e.g. *`/ATmega328_UART/examples/IO_streams/Makefile`*)_ and edit the variables: `$HOST_PLATFORM`, `$F_CPU`, `$PROGRAMMER` and `$PORT`.
48  - Execute `> make` inside the same directory to build the example.
49  - And finally execute `> make program` to upload it to the AVR target.
50 
51 #### Building and uploading a new project using one of the example's *`Makefile`*
52 This is the same as ['Building and uploading an example'](#building-and-uploading-an-example) except that the *`Makefile`* should be configured for the new project _(e.g. the target name, the new location of the library, etc)_.

56 ### Building and uploading manually :wrench:
57 The library can be compiled and linked to a project using a different build system _(e.g. a user's *`Makefile`*, an IDE, etc)_. The location of the header file is *`.../ATmega328_UART/include/ATmega328_UART.h`* and the location of the source file is *`../ATmega328_UART/src/ATmega328_UART.c`*.
58 
59 > _`F_CPU` and `BAUD` macros must be defined._
60 
61 > :grey_exclamation: _The library must be recompiled every time `F_CPU` and/or `BAUD` macros are changed!_
62 
63 
64 > When using formatted IO streams, the libraries *printf* and/or *scanf* should be linked[<sup>1</sup>](#1). AVR-GCC linker flags:
65 > - `-Wl,-u,vfprintf -lprintf_min`
66 > - `-Wl,-u,vfprintf -lprintf_flt -lm`
67 > - `-Wl,-u,vfscanf -lscanf_min`
68 > - `-Wl,-u,vfscanf -lscanf_flt -lm`
